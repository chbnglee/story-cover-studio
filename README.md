# Story Cover Studio

3:4 세로 동화 표지를 Gemini 이미지 모델에 참조 이미지로 보내 16:9 표지와 배너 3종을 각각 재생성하고, 플랫폼용 규격으로 저장하는 로컬 웹앱입니다.

## 필요 항목

- Node.js LTS
- Gemini API key

## 실행

1. `run.bat`를 더블클릭합니다.
2. Gemini API key를 붙여넣습니다.
3. 브라우저가 자동으로 열립니다.
4. 자동으로 열리지 않으면 콘솔에 표시된 `http://localhost:포트번호` 주소를 Chrome 또는 Edge에서 엽니다.

기본 포트는 `5173`입니다. 이미 사용 중이면 앱이 `5174`, `5175`처럼 다음 빈 포트를 자동으로 사용합니다.

`index.html`을 직접 더블클릭하면 프로필 크롭 같은 브라우저 기능은 보일 수 있지만, Gemini 재생성은 동작하지 않습니다.

## 표지 작업 흐름

1. `3:4 표지 업로드`에 세로 표지를 넣습니다.
2. `Story ID`에 저장 파일명의 기준이 될 ID를 입력합니다.
3. 각 규격 카드의 `생성` 버튼을 눌러 1920x1080 표지와 배너 3종을 각각 새로 만듭니다.
4. 각 규격 카드의 `저장` 버튼으로 개별 저장하거나, `생성본 전체 저장`을 누릅니다.
5. 필요한 경우 크롭 위치와 최종 확대 값을 미세 조정합니다.
6. `현재 규격 저장`, `배너 3종 저장`, 또는 `전체 저장`을 누릅니다.

## 출력 규격

- 16:9 표지: `1920 x 1080`
- 커버 배너 L: `1332 x 404`
- 커버 배너 M: `814 x 262`
- 커버 배너 S: `560 x 207`
- 원형 프로필 PNG: `600 x 600`, 원 밖 투명 배경, 테두리 없음

## 생성 방식

- 모든 생성은 업로드한 3:4 표지를 참조 이미지로 사용합니다.
- 배너 3종은 서로 다른 Gemini 요청으로 생성됩니다.
- 각 배너 프롬프트에는 해당 목표 픽셀, 목표 비율, 제목 안전영역, 캐릭터/소품 보존 조건이 별도로 들어갑니다.
- 프롬프트에서 단순 리사이즈, 크롭, 늘리기, 패딩, 흐린 배경 확장, 원본 표지 삽입을 금지합니다.
- Gemini가 반환한 이미지는 마지막 단계에서만 정확한 픽셀 규격으로 크롭 또는 리샘플링됩니다.

## API key 보관

API key는 파일에 저장하지 않습니다. `run.bat` 실행 중에만 환경 변수로 전달됩니다.
